# Security Policy

## Supported Versions

This project is an early prototype. Only the current `main` branch is supported for security review.

## Reporting a Vulnerability

Do not open public GitHub issues for security vulnerabilities.

Contact the repository owner privately through GitHub first. Include a short summary, affected files or routes, reproduction steps, and the expected impact.

## Testing Rules

This repository is proprietary. Public source visibility does not grant permission to perform security testing against any deployed instance, hosted service, model endpoint, database, camera feed, or private network.

Do not run vulnerability scans, load tests, credential attacks, camera probing, RTSP enumeration, or automated exploit attempts without prior written permission.

## Disclosure

Please give the owner reasonable time to investigate and fix confirmed issues before sharing details with anyone else.
